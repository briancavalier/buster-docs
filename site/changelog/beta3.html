Buster.JS Beta 3

Breaking changes

This is a list of breaking changes in this release. Since we haven't reached 1.0 stable yet, we're changing APIs without making them backwards compatible.

Some expectations changed names

We're renaming some of our expectations, the intent is basically to match the expectations in Jasmine since that's what people are used to, and since they have good well thought out names.

Changes

  • Fixed some bugs in server proxying for browser tests (#57)
  • Added a new property to the config file, testHelpers. It's identical to tests, except that they will not be excluded when running specific test files with buster test -t tests/my-test.js. #95
  • The body of the testbed HTML in browser tests will now reset between each test run. It will not be cleared out entirely, it will be set to what it was initially. Note: this is not yet fixed in buster-static. (#74)
  • Bug: Browser tests now fail when a test times out when there are successful tests in the same test run. (#77)
  • Bug: Browser tests now fail when there's no assertions in a test. (#69)
  • Bug: buster.log(function(){}); would log undefined, as it called the function because of internals in buster-evented-logger. It no longer calls the function, and logs what you'd expect it to log. (#94)
  • Bug: Asserts are now counted properly in the JsTestDriver extension. (#49, #31)
  • At some point in time, an unknown change fixed a small problem with assert.calledOnce. Nobody knows what, where and why. (#70)
